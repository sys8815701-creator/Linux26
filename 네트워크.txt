[네트워크]

-------------------------------------------------------------------------------
  1. 네트워크 종류
-------------------------------------------------------------------------------

  ▶ URL 구조
  -------------------------
    형식 : 프로토콜://IP주소:포트번호

    숫자 주소와 영문 주소
      - DNS를 통해 도메인 ↔ IP 주소 1:1 변환 (bind)
        e.g. mbc-sw.iptime.org → 61.79.123.146

    주요 프로토콜 및 기본 포트
      - http://   웹 서비스          기본 포트 :80
      - https://  보안 웹 (SSL/TLS)  기본 포트 :443
      - ftp://    파일 전송          기본 포트 :21
      - smtp://   이메일 발신        기본 포트 :25 (보안 587)

    ※ 포트 : 서비스와 연결되는 통로 (내선번호 개념)

  ▶ 포트 번호 범위
  -------------------------
    - 알려진 포트 (0 ~ 1023) : IANA 공식 등록, 시스템 권한 필요
    - 알려지지 않은 포트 (1024 ~ 65535) : 자율적으로 서비스 사용 가능

-------------------------------------------------------------------------------
  2. IP 주소 체계 : IPv4, IPv6
-------------------------------------------------------------------------------

  ▶ IPv4 (32bit)
  -------------------------
    범위 : 0.0.0.0 ~ 255.255.255.255

    -- 주소 고갈 해결 방법 : 사설망과 공인망 분리 --

    [사설망] 공유기(NAT) 내부에서 사용하는 IP (본인이 만들어 사용)
      A 클래스 : 10.0.0.0    ~  10.255.255.255
      B 클래스 : 172.16.0.0  ~  172.31.255.255
      C 클래스 : 192.168.0.0 ~ 192.168.255.255

    [공인망] 전 세계적으로 단일 장비에 할당되는 고유 IP

    공인망 ↔ 사설망 연결 장비 : 라우터 / IP 공유기 / NAT

    -- 도커(Docker) --
      - IP 1개를 포트별로 나누어 컨테이너에 할당
      - 컨테이너 : 핵심 프로그램을 묶어 독립적으로 서비스하는 실행 단위

  ▶ IPv6 (128bit)
  -------------------------
    표기 : 16진수(0~F) 8그룹, 콜론으로 구분
     e.g. 2001:0db8:85a3:0000:0000:8a2e:0370:7334

    연속되는 0 필드는 ::으로 축약 가능
     e.g. 2001:0db8:85a3::8a2e:0370:7334

    ※ IPv4 주소 고갈 문제의 근본적 해결책
